# Кадровые документы: два канала доставки

Рабочий проект по цели «Аналитическое заключение по направлению кадровых документов работникам в электронном формате и Почтой России с электронным уведомлением о вручении».

**Сайт:** https://trainingsw75-cloud.github.io/kedo-dva-kanala/

## Что внутри

| Файл / папка | Что это |
|---|---|
| `index.html` | Главная страница — «полочки»: каждый раздел нажимается и раскрывается |
| `prezentaciya.html` | Версия-презентация: 10 слайдов с навигацией стрелками |
| `qr.html` | QR-инструмент: код отслеживания отправления по трек-номеру или код с любой ссылкой; формируется в браузере, печать в один клик. Понимает параметры `?track=…&cap=…` |
| `zhurnal.html` | Журнал отправлений: учёт писем (работник, документ, трек, статус), QR и отслеживание из строки, экспорт CSV. Данные — только в localStorage браузера |
| `soglasiya.html` | Реестр согласий на КЭДО: статус по каждому работнику (дано / не требуется / отказ / отозвано), автоматический канал А/Б, поиск, экспорт CSV. Данные — только в localStorage браузера |
| `vendor/qrcode.js` | Библиотека генерации QR-кодов (qrcode-generator, © Kazuhiko Arase, MIT) |
| `documents/` | Исходные документы: отчёты, аналитическая записка, письмо в ЮС, бланки согласия (RTF), схема и презентация (PDF) |

## Разделы сайта

1. Итоговый отчёт: цель достигнута (4 критерия)
2. Как выбрать канал: два вопроса-маршрутизатора
3. Канал А — КЭДО (электронный)
4. Канал Б — Почта России + ЮЗЭУВ (бумажный)
5. Зона внимания: ЭЗП — отдельный продукт
6. Согласие работника: три бланка и четыре правила
7. Шесть ключевых выводов анализа
8. QR-коды в процессе (не замена описи ф. 107 — навигация к доказательствам)
9. Документы для скачивания

## Как обновлять

Сайт статический (GitHub Pages), правки — в HTML-файлах. После пуша в ветку `main` сайт обновляется автоматически за 1–2 минуты.

Основания: ст. 22.1–22.3 ТК РФ, открытые материалы АО «Почта России», судебная практика об описи вложения. Материал не заменяет позицию юридической службы.
